J2EE - Java Enterprise edition

Yleistä

Tämän sivun tarkoituksena on tarjota alkeellinen esimerkki siitä, kuinka Java-ohjelmien ja verkkosivujen kokonaisuus ladataan sovelluspalvelimelle.

Ympäristön pystyttäminen

Koska tässä esimerkissä ei käytetä tietokantaa, riittää pelkkä sovelluspalvelin. Valitaan sovelluspalvelimeksi

  • Apache Tomcat
Ladataan sieltä uusimman version (kirjoitushetkellä 6.0.18) zip-paketti ja puretaan se kotihakemiston alle tehtyyn hakemistoon (esim. "Application servers"): 
$ unzip apache-tomcat-6.0.18.zip
Tämän jälkeen varmistetaan, ettei mikään muu sovelluspalvelin ole päällä menemällä osoitteeseen http://localhost:8080/ ja jos sieltä tulee vastaukseksi virhe, niin kyseinen portti on ilmeisesti vapaa. Käynnistetään Tomcat: 
$ cd apache-tomcat-6.0.18/bin
$ chmod 755 *.sh
$ ./startup.sh
jolloin ruudulla pitäisi lukea suunnilleen: 
Using CATALINA_BASE: ...
Using CATALINA_HOME:
Using CATALINA_TMPDIR: ...
Using JRE_HOME: ...
Jos näin ei tapahdu, on todennäköisenä syynä se, ettei Java Development Kit:iä (Javan kehitysympäristö) tai Java Runtime Environment:iä (Javan ajoympäristö) ole oikein asennettu. Nämä on mahdollista ladata ja asentaa Sun Microsystems:in sivulta. Jos Tomcat sitävastoin on käynnistynyt normaalisti, niin menemällä nyt uudestaan osoitteeseen http://localhost:8080/, pitäisi näkyviin tulla Apache Tomcat:in oletussivu. Tarkastetaan edelleen, että palvelimen sulkeminen onnistuu normaalisti, eli ajamalla 
$ ./shutdown.sh
pitäisi Tomcatin sivun lakata näkymästä kun sille yrittää mennä uudelleen.

Tomcatin käyttö

Kun Tomcatin käynnistää uudelleen, löytyy sen sivulta itsessään jo runsaasti ohjeita palvelimen käyttöön. Erityisen suositeltavaa on mennä ensiksi tälle paikalliselle sivulle ja klikata sen esimerkit läpi. Hyötynä tästä on se, että omaa sovellusta varten löytyy helposti "pohjakoodi".
